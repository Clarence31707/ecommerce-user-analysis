电商用户增长与精细化运营分析

基于 Olist Brazilian E-Commerce 公开数据集，通过 MySQL 数据处理、指标体系建设、RFM 用户分层及 Power BI 可视化，完成从数据清洗到运营策略输出的完整分析闭环。

分析周期：2016.09 - 2018.08
项目背景：Olist 是巴西本土综合电商平台，在业务扩张期面临新客成本上升、老客复购不足的挑战。本项目旨在通过数据驱动的方式定位增长瓶颈，并输出可落地的用户精细化运营策略。

技术栈：MySQL  Power BI  Navicat

核心分析与产出
数据清洗与质量管控：构建订单粒度宽表：关联 6 张核心业务表，处理约 9.6 万笔订单。
主动修正关键口径问题：发现 订单 x 商品 粒度导致 GMV 重复累加虚高 28% 的问题，重构为订单粒度。
识别 customer_id 为技术主键，改用 customer_unique_id 还原被掩盖的复购行为，成功识别出 1284 名复购用户。
指标体系搭建：建立覆盖 GMV、订单量、客单价、新增用户、MAU、复购率等 10+ 核心指标的月度经营指标体系，通过时间序列追踪平台增长趋势。
核心增长诊断：精准定位平台核心瓶颈：整体用户复购率仅 3.16%，增长过度依赖新客驱动。
RFM 用户精细化分层：模型优化：对比等频分箱（NTILE）与业务阈值分箱效果，选用业务阈值法解决评分失真的问题，使分层结果更具业务区分度。
分层结果：将 9.5 万用户划分为 4 类（高价值、潜力、流失风险、沉睡），发现沉睡用户占比高达 67%，存在巨大存量召回空间。
运营策略与价值预估：针对不同层级用户输出差异化运营方案（沉睡召回、潜力复购引导、高价值维护），测算召回策略可带动 GMV 增长约 30 万元。
可视化看板搭建：使用 Power BI 搭建 经营概览、用户增长、用户价值 3 页交互式看板，实现业务数据动态监控。

关键发现
高价值用户极度稀缺：仅 8人（0.01%），但人均消费是平台均值的 3-10 倍，是平台最核心的利润贡献者。
沉睡用户体量庞大：67% 的用户超过 180 天未消费，是低成本召回的首要目标。
复购是核心瓶颈：97% 的用户仅消费一次，首单后流失严重，提升复购率是突破增长瓶颈的关键。
